ヘッドレス WordPress 向けの任意の期限付きプレビュートークンを発行します。指定したフロントエンド環境 URL で直接プレビューを開き、ドラフトコンテンツを取得できます。

== 説明 ==

**Preview Token** は、デカップル型（ヘッドレス）WordPress 構成における認証問題を解決します。こうした用途で使用可能な Application Password は WordPress に標準機能として備わっている素晴らしい仕組みですが、長期的なシークレットをフロントエンド側で管理する必要が生じます。代わりにこのプラグインでは、投稿ごとに読み取りアクセスを期限付きで許可するトークンを発行します。

フロントエンド（Astro、Next.js、Nuxt など）はプレビュー URL を受け取り、REST API 経由でドラフトコンテンツをダイレクトに取得できます。永続的な認証情報の管理は不要です。

= 仕組み =

1. 権限のある WordPress ユーザーが Gutenberg サイドバー、クイック編集パネル、またはクラシックエディターのメタボックスからトークンを生成します。
2. トークンが外部フロントエンドを指す URL に埋め込まれます。
3. フロントエンドが `/wp-json/preview-token/v1/preview?token=…` を呼び出してドラフトコンテンツを取得します。
4. トークンは自動的に期限切れとなり、手動でのクリーンアップは不要です。

= 主な機能 =

**トークン管理**

* 有効期限プリセット：1時間・24時間・30日・カスタム日時・無期限。
* 1投稿につきトークンは1件のみ — 新しいトークンを発行すると既存のトークンが無効化されます。
* トークンは `bin2hex(random_bytes(32))`（256-bit CSPRNG）で生成。`wp_options` に保存される検索キーは raw トークンではなくその SHA-256 ハッシュ値です。DB が漏洩しても有効なトークン文字列が直接露出しません。
* WP Cron による期限切れトークンの自動クリーンアップ（毎日）。

**エディター統合**

* **Gutenberg**：ドキュメント設定サイドバーに専用パネルを表示。
* **クイック編集**：投稿一覧画面から直接トークンを操作。
* **クラシックエディター**：エディターサイドバーにメタボックスとして表示。
* ワンクリックでプレビュー URL をクリップボードにコピー。
* トークンを再生成せずに有効期限のみ更新可能。

**管理画面設定**

* 外部フロントエンドの URL を設定（「外部プレビューを開く」をクリックすると、この URL へ直接遷移します）。
* 許可する CORS オリジンを複数設定（ワイルドカード対応 — `https://*.example.com`）。
* トークン発行に必要な最低 WordPress ロールを選択（購読者〜管理者）。
* レート制限を調整（時間ウィンドウあたりのリクエスト数）。
* 無期限トークンの発行を任意で許可。

**発行済みトークン一覧**

* すべてのアクティブ・期限切れトークンを投稿タイトル・ステータス・有効期限・発行者とともに表示。
* 個別の失効、または期限切れの一括削除を設定画面から実行可能。

**セキュリティ**

* プレビューエンドポイントは HTTPS が必要（ローカル開発時は無効化可能）。
* IP アドレス単位のレート制限（デフォルト：60秒あたり30リクエスト）。
* ロールベースのアクセス制御。
* 明示的に設定したオリジンにのみ CORS ヘッダーを送信。WP コアの無条件エコーバックを上書き。
* `Referrer-Policy: no-referrer` によるリファラー経由のトークン漏洩防止。
* `draft`・`pending`・`future` ステータスの投稿のみプレビュー対象。
* 管理者専用の設定ページ（多層的な権限チェック）。
* すべての管理アクションに CSRF 対策（nonce 検証）。

**監査ログ**

* トークンの発行・使用イベントをログ記録（投稿 ID・ユーザー ID・クライアント IP）。
* セキュリティイベントのログ記録：無効なトークン試行・レート制限超過・権限拒否。
* デフォルトは `WP_DEBUG_LOG` に出力。`PVT_LOG_FILE` で専用ファイルに変更可能。

**国際化対応**

* 日本語（ja）・簡体字中国語（zh_CN）翻訳を同梱。
* すべての管理画面 UI 文字列が翻訳対応済み。

= 開発者向けフック =

**フィルター**

* `pvt_preview_response_data` — 送信前に REST API レスポンスデータを変更します。

**アクション**

* `pvt_token_issued( int $post_id, int $user_id )` — トークン発行後に発火。
* `pvt_token_used( int $post_id, int $user_id )` — トークンが正常に使用されたときに発火。
* `pvt_invalid_token( string $ip )` — 無効・期限切れトークンのアクセス時に発火。
* `pvt_rate_limit_exceeded( string $ip, string $endpoint )` — レート制限超過時に発火。
* `pvt_capability_denied( int $user_id, int $post_id )` — 権限拒否時に発火。

**定数（wp-config.php）**

* `PVT_SKIP_HTTPS_CHECK` — `true` に設定すると HTTPS 要件を無効化します（開発環境専用）。
* `PVT_LOG_FILE` — 専用監査ログファイルの絶対パス。

= ユースケース =

このプラグインは、Astro・Next.js・Nuxt・SvelteKit などの外部フロントエンドが WordPress REST API からコンテンツを取得する**ヘッドレス WordPress** 構成向けに設計されています。WordPress アカウントを付与したり長期的な認証情報を公開したりすることなく、コンテンツ編集者がステークホルダーとドラフトプレビューを安全に共有できます。

== インストール ==

1. `preview-token` フォルダを `/wp-content/plugins/` にアップロードします。
2. WordPress の**プラグイン**メニューからプラグインを有効化します。
3. **設定 → Preview Token** で**外部プレビュー URL**（フロントエンドのベース URL）を設定します。
4. フロントエンドドメインの許可 CORS オリジンを追加します。
5. Gutenberg サイドバー、クイック編集パネル、またはクラシックエディターのメタボックスからトークンを生成します。

= JavaScript ソースコード =

`assets/js/` 内のファイルはコンパイル済みのバンドルです。WordPress.org ガイドラインに基づき、人間が読める TypeScript ソースファイルは以下で公開しています：

https://github.com/uuki/preview-token/tree/main/src/js

= ソースからのビルド =

TypeScript ソースから JavaScript バンドルを再ビルドするには：

1. リポジトリをクローン：`git clone https://github.com/uuki/preview-token.git`
2. Node.js 依存関係をインストール：`pnpm install`
3. コンパイル：`pnpm run build`

ビルド設定は `tsdown.config.ts` に定義されています。`assets/js/` 内の各バンドルは `src/js/` のエントリポイントに対応しています。

= 最低動作環境 =

* WordPress 5.9 以降
* PHP 7.4 以降
* WordPress REST API を利用できる外部フロントエンド

== よくある質問 ==

= フロントエンドは特定のフレームワークである必要がありますか？ =

いいえ。WordPress REST API を呼び出せる HTTP クライアントであれば何でも動作します。プラグインは標準的な WordPress REST API の投稿オブジェクトを返します。

= プレビューできる投稿ステータスは何ですか？ =

`draft`（下書き）・`pending`（レビュー待ち）・`future`（予約済み）のみです。公開済みの投稿は意図的に除外されています（すでに公開されているため）。

= 同じプレビュー URL を複数人で使えますか？ =

はい。トークンは有効期限内であれば繰り返し使用できるよう設計されています（例：プレビューの再読み込み、異なる画面サイズでの確認）。新しいトークンを発行すると、以前の URL は無効になります。

= トークンが期限切れになるとどうなりますか？ =

トークンは 401 レスポンスで拒否されます。期限切れトークンは WP Cron の日次ジョブによって自動的に削除されます。エディター UI では、有効期限の状態を表示せずにトークン生成画面が表示されます。

= HTTPS は必須ですか？ =

デフォルトでは必須です。トークンが通信経路上で傍受されるのを防ぐため、プレビューエンドポイントは HTTP リクエストに 403 を返します。ローカル開発環境では、`wp-config.php` に `define('PVT_SKIP_HTTPS_CHECK', true);` を追加することでバイパスできます。

= トークンを発行できるユーザーを制限するには？ =

**設定 → Preview Token → 最低ロール** で最低 WordPress ロールを選択します（購読者・寄稿者・投稿者・編集者・管理者）。そのロール未満のユーザーがトークンを発行しようとすると 403 が返されます。

= CORS 設定でワイルドカードを使えますか？ =

はい。`https://*.example.com` のようなパターンでドメインの全サブドメインを許可できます。ベアワイルドカード `*` も対応していますが、セキュリティ警告が表示されます — 可能な限り具体的なパターンを指定してください。

= 監査ログはどこに保存されますか？ =

デフォルトでは、ログは PHP の `error_log()` を介して出力されます（`WP_DEBUG_LOG` の設定に従います）。専用ファイルに書き出すには、`wp-config.php` に `define('PVT_LOG_FILE', '/絶対/パス/pvt.log');` を追加してください。

= クラシックエディタープラグインと互換性がありますか？ =

はい。クラシックエディタープラグインが有効な場合、トークンパネルが投稿エディターのサイドバーにメタボックスとして表示されます。

== スクリーンショット ==

1. **Gutenberg サイドバー** — トークン生成前のパネル。有効期限を選択して「トークンを生成」をクリックします。
2. **Gutenberg サイドバー** — アクティブなトークン：有効期限・指定フロントエンド URL でのプレビュー・URL コピー。
3. **クラシックエディター** — メタボックスサイドバーのトークンパネル（生成前）。
4. **クラシックエディター** — アクティブなトークンと「外部プレビューを開く」ボタン。
6. **設定ページ** — フロントエンド URL・CORS オリジン・最低ロール・レート制限を設定します。
7. **発行済みトークンタブ** — 投稿・ステータス・有効期限・発行者を一覧表示。個別または一括で失効させます。
8. **クイック編集パネル** — エディターを開かずに投稿一覧画面からトークンを管理します。
